## 2. 화면 레이아웃 (고정, 임의 변경 금지)

```
┌─────────────────────────────────┐
│ 라운드 N    ⏱ 04:18    💎 350 🟢 125 👤 24/200 │  ← HUD (게임화면 위 오버레이, 배경 없음)
│ 킬 24                           │
│                                 │
│                                 │
│         게임 화면                │  ← 게임 월드 (항상 실제 맵 렌더링)
│       (터치 인터랙션)             │     절대 UI 리스트/그리드 없음
│                                 │
│         📍 내 디펜스 구역         │  ← 현재 구역 레이블 (하단 고정)
├─────────────────────────────────┤
│ [메인] [유닛뽑기] [업그레이드] [플레이어] │  ← 탭바
│                                 │
│         하단 커맨드 카드          │  ← 탭/선택 상태에 따라 변화
│                                 │
└─────────────────────────────────┘
```

### 2-1. 비율
- 게임 화면: 전체 높이의 약 70% (flex: 1, 하단 패널이 고정 높이를 가져간 나머지)
- 하단 패널: 고정 높이 220px (탭바 44px + 컨텐츠 176px)
- 스크롤 없음. 모든 요소가 한 화면에 표시되어야 함

### 2-2. HUD 오버레이 (배경 레이어 없음, 텍스트만)
| 위치 | 내용 |
|------|------|
| 좌상단 (세로 나열) | 라운드 N / 킬 N |
| 중앙 상단 | ⏱ 시간 (MM:SS) |
| 우상단 (가로 나열, 한 줄) | 💎 미네랄 값 / 🟢 가스 값 / 👤 현재/최대 인구수 |

- HUD는 게임 화면 위에 position: absolute로 띄움
- 텍스트에 진한 drop-shadow(여러 겹) 처리하여 어두운 배경·밝은 배경 모두 가독성 확보
- 맵에 따라 표시 항목이 다를 수 있음 (라운드 대신 웨이브, 킬 대신 점수 등 — 추후 확장)

## 3. 게임 화면 — 탭별 뷰

**핵심 규칙: 탭을 바꾸면 게임 화면 전체가 해당 구역 뷰로 전환된다.**
게임 화면에는 절대 UI 리스트, 카드 그리드 등을 렌더링하지 않는다.
게임 화면 = 항상 맵/지형 + 그 위의 게임 오브젝트(유닛, 건물, 적 등).

### 3-1. 메인 탭 — 디펜스 구역
- 사각형 트랙과 적, 내 유닛들이 보이는 실제 플레이 화면
- 유닛은 맵 어디에나 자유 배치 (고정 슬롯 없음)
- 유닛 터치 → 해당 유닛 선택 (하단 커맨드 카드 변경)
- 맵 빈 곳 터치 → 선택 해제 (하단 커맨드 카드 기본 상태로)

### 3-2. 유닛뽑기 탭 — 생산 구역
- 게임 화면에 생산 건물(배럭스, 팩토리, 스타포트 등)이 맵 위에 배치되어 있음
- 건물 터치 → 하단 커맨드 카드에 해당 건물의 생산 가능 유닛 목록 표시
- 유닛 버튼 터치 후 게임 화면의 원하는 위치로 드래그 → 해당 위치에 유닛 배치

### 3-3. 업그레이드 탭 — 테크 구역
- 게임 화면에 테크 건물(공학 베이, 과학 시설, 아머리, 아카데미 등)이 배치되어 있음
- 건물 터치 → 하단 커맨드 카드에 해당 건물의 업그레이드 목록 표시
- 업그레이드 버튼 터치 → 비용 차감 후 연구 시작

### 3-4. 플레이어 탭 — 다른 플레이어 화면
- 게임 화면에 현재 선택된 플레이어의 실제 플레이 화면이 표시됨 (그 사람의 디펜스 구역, 유닛, 적 등)
- 플레이어 전환 버튼은 **하단 커맨드 카드 영역**에 위치 (게임 화면 위에 오버레이 하지 않음)

## 4. 하단 커맨드 카드 — 상태별 명세

### 4-1. 메인 탭 / 기본 상태 (아무것도 선택 안 됨)
```
[ 이동 ] [ 선택 ] [ 조합 ] [ 상태창 ] [ 배속 ]
[ 미니맵 ] [ 일시정지 ] [ 설정 ]
```

### 4-2. 메인 탭 / 유닛 선택 상태
```
[ 유닛 초상화 | 유닛명 + 스탯 ]   [ ✕ 해제 ]
[ 부대 선택 ] [ 지정 해제 ] [ 이동 ] [ 위치 고정 ]
```
- 유닛 초상화: 해당 유닛 이미지(또는 이모지 placeholder)
- 유닛명, 공격력, 사거리 표시
- 버튼 추가 가능 구조로 설계 (나중에 확장)

### 4-3. 유닛뽑기 탭 / 건물 선택 전
```
"건물을 터치해서 구매할 유닛을 선택하세요"  (안내 텍스트)
```

### 4-4. 유닛뽑기 탭 / 건물 선택 후
```
[ 유닛A 아이콘+이름+비용 ] [ 유닛B ] [ 유닛C ] ...
```
- 건물에 따라 버튼 구성이 다름

### 4-5. 업그레이드 탭 / 건물 선택 전
```
"건물을 터치하면 업그레이드 목록이 표시됩니다"  (안내 텍스트)
```

### 4-6. 업그레이드 탭 / 건물 선택 후
```
[ 업그레이드명 ] [ 비용 ] [ 연구 버튼 ]
[ 업그레이드명 ] [ 비용 ] [ 연구 버튼 ]
...  (스크롤 가능)
```

### 4-7. 플레이어 탭
```
[ P1 (나) ] [ P2 ] [ P3 ] [ P4 ]
```
- 현재 내 화면이면 P1 강조 표시
- 플레이어 수는 맵에 따라 달라짐 (추후 설정 가능하도록)

## 5. 게임 메커니즘 — 네모네모 디펜스

### 5-1. 맵 구조
```
  S(스폰) ──────────────────→
     │                      │
     ↑    (유닛 자유 배치)    │
     │                      ↓
     ←──────────────────────
```
- 사각형 트랙: 가장자리에서 INSET만큼 떨어진 정사각 둘레가 적의 이동 경로
- INSET = 게임 화면 너비의 12%
- 적은 좌상단(S)에서 스폰, 시계방향 순환
- 트랙은 루프 — 끝 없음. 적은 처치될 때까지 계속 돔
- `posAt(d)` 함수: 둘레 거리값 d → {x, y} 좌표 변환

### 5-2. 유닛 배치
- 유닛은 트랙 안쪽 영역 어디에나 자유 배치 가능
- 고정 슬롯 없음
- 유닛은 사거리 내 적을 자동 공격
- 선택된 유닛은 이동 명령으로 위치 변경 가능

### 5-4. 합성 (조합)
- 같은 종류 + 같은 등급의 유닛 2기 → 한 등급 상위 유닛 1기로 합성
- 조합 버튼 탭 → 합성 모드 진입 → 유닛 탭 → 유닛 탭 → 자동 합성

```
src/

      DefaultCard.tsx   ← 기본 버튼들
```
