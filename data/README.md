# 종족 / 유닛 기본 설정 (data)

스타크래프트 유즈맵(모바일)의 **기본 데이터**입니다. 게임 로직과 분리되어 있어, 새 유즈맵을 만들 때 이 파일을 불러와서 그대로 쓰거나 일부 값만 덮어써서 사용합니다.

## 파일
- `races.json` — 3종족과 종족별 유닛(총 34), 코스트/스탯/특성 정의
- `heroes.json` — 종족별 영웅 유닛 (기본 유닛 기반 강화형)
- `economy.json` — 자원 채취(미네랄/가스) · 인구(보급) 시스템 규칙
- `tech.json` — 종족별 건물(테크 트리) · 업그레이드 · 특수능력 해금
- `*.schema.json` — 각 데이터의 구조 검증/자동완성용 스키마

데이터는 모두 게임 로직과 분리되어 있어, 새 유즈맵에서 `import` 후 일부 값만 덮어써서 사용합니다.

## 종족 구성
| 종족 | 테마 | 특징 |
|------|------|------|
| 테란 (terran) | 인간 | 균형형, 기계 중심, 수리 가능 |
| 프로토스 (protoss) | 외계 | 고비용·고성능, 전 유닛 보호막(실드) 보유 |
| 저그 (zerg) | 괴물 | 저비용 물량, 자가 회복(리젠) |

## 코스트 기준
- `minerals` 미네랄 / `gas` 가스 / `supply` 보급(인구수)
- 원작 스타크래프트 값을 기반으로, 모바일 단순화를 위해 일부만 조정

## 스탯 필드
| 필드 | 설명 |
|------|------|
| hp | 체력 |
| shield | 보호막 (프로토스만 사용) |
| armor | 방어력 |
| damage | 공격력 |
| attackType | `normal`(일반) / `explosive`(폭발-소형에 약함) / `concussive`(진동-대형에 약함) / `none`(비공격) |
| range | 사거리 (1=근접, 2+=원거리, 타일 단위) |
| sight | 시야 |
| speed | 이동 속도 |
| size | `small`/`medium`/`large` (데미지 보정용) |
| regen | 초당 체력 자동 회복량 (주로 저그) |

## 유닛 특성 플래그 (선택 필드)
| 필드 | 의미 |
|------|------|
| detector | `true`면 은신 감지 가능 (테란 사이언스 베슬, 프로토스 옵저버, 저그 오버로드) |
| cloak | `true`면 은폐 가능 (고스트·레이스·다크템플러·러커·옵저버 등) |
| transport | 숫자 = 수송 칸 수 (드랍십/셔틀/오버로드) |
| providesSupply | 보급 제공량 (오버로드=8) |
| mergeFrom | 합체/변태 재료 유닛 id (아콘=하이템플러×2, 가디언=뮤탈, 러커=히드라) |

## 종족 특성 (race.traits)
- **테란**: 수리(repair) — SCV로 기계/건물 수리, 메딕으로 생체 치료
- **프로토스**: 보호막 자동 회복 (shieldRegenRate)
- **저그**: 체력 자동 회복 (regenRate), 라바 변태 생산

## 데미지 보정 규칙 (원작 기반, 권장)
| 공격 타입 \ 크기 | small | medium | large |
|------------------|-------|--------|-------|
| normal | 100% | 100% | 100% |
| explosive | 50% | 75% | 100% |
| concussive | 100% | 50% | 25% |

## 사용 예시
```js
import races from "./data/races.json";

// 테란 마린의 코스트
const marine = races.races.terran.units.find(u => u.id === "marine");
console.log(marine.cost); // { minerals: 50, gas: 0, supply: 1 }

// 특정 유즈맵에서 마린 체력만 강화 (override)
const customMarine = { ...marine, stats: { ...marine.stats, hp: 80 } };
```

## 테크 트리 / 자원 (요약)
- `tech.json`: 본진 → 생산 건물 → 상위 건물 순으로 `requires`로 연결. 각 건물의 `unlocks`가 생산 가능한 유닛, `upgrades`/`abilities`의 `appliesTo`가 races.json의 유닛 id와 매칭됨.
- `economy.json`: 미네랄/가스 1회 채취 8, 보급 건물당 +8, 보급 상한 200. 모바일 단순화 옵션은 `mobileSimplification` 참고.

## ⚠️ 지적재산(IP) 주의 — 꼭 읽기
스타크래프트는 블리자드의 IP입니다. 현재 데이터는 **개발·학습용 밸런스 레퍼런스**로 원작 이름을 그대로 사용합니다.

| 요소 | 상태 |
|------|------|
| 수치·코스트·게임 규칙 | ✅ 메커니즘이라 비교적 안전 (그대로 사용 OK) |
| 이름(테란/마린 등)·세계관·디자인·원작 에셋 | ⚠️ 블리자드 IP — **공개/배포 전 반드시 교체** |

### 공개 결정 시 리브랜딩 방법 (1파일 작업)
- `id`(예: `marine`)는 **코드용 고정 키** → 절대 바꾸지 말 것 (로직이 참조)
- `name`(예: "마린")은 **화면 표시용** → 이 값만 자체 명칭으로 교체
- 종족 `name`/`theme`, 유닛 `name`, 건물 `name`만 바꾸면 밸런스·로직은 그대로 유지됨
- 원작 그래픽/사운드 에셋은 사용하지 말고 직접 제작하거나 라이선스된 것 사용
