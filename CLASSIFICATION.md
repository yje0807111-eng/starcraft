# 유즈맵 공통/전용 분류표 (Classification v1)

> 목적: 여러 유즈맵을 한 앱에서 운영하기 위해, **모든 유즈맵이 공유하는 공통 프레임워크(셸)** 와
> **각 유즈맵이 개별로 정의하는 요소(per-map)** 를 분류한다. 기준은 현재 유일하게 구현된
> **네모네모 디펜스** 다. 이 문서가 이후 구현·리팩터의 기준이 된다.
>
> 상태: 분류 합의 완료(v1). 코드 리팩터는 아직 미착수.

## 🟢 공통 프레임워크 (모든 유즈맵이 공유 = 셸)

| 영역 | 내용 | 비고 |
|---|---|---|
| 부팅/인증 | 오프닝 로딩 · 로그인/회원가입(Supabase) · 자동 로그인 · 닉네임 | |
| 메뉴 | 유즈맵 선택 · 모드 선택(개인/멀티) | |
| 멀티 | 방 찾기 · 방 생성 · 로비(대기실) · 채팅(로비/게임) | |
| **자원 시스템** | 미네랄 · 에너지 · 인구 **구조 공통** | **수급량/배율만 맵별 조정** |
| **업그레이드** | 패널 · 레벨 · 비용 · 적용 **골격 공통** | **항목 목록은 맵별 교체** |
| UI 셸 | 폰 프레임 · 우주 배경 · 검정+빨강 테마 · 토스트 · 나가기 확인 | |
| HUD 상단바 | 바 **골격은 공통** | 표시 요소(라운드·타이머 등)는 맵별 on/off |
| 하단 탭바 | 탭 **바/전환 골격은 공통** | 탭 구성 자체는 맵별(아래) |
| 설정 팝업 | 소리/비디오/배속/일시정지/나가기 골격 | 기능 on/off는 맵별 |

## 🔴 유즈맵별 정의 (per-map)

| 영역 | 내용 | 결정 |
|---|---|---|
| **탭 구성** | 탭 개수·종류를 맵마다 자유롭게 정의 (예: ‘마린 키우기’는 유닛뽑기 대신 ‘상점’) | 맵별 자유 ✅ |
| 게임플레이 | 지형/보드 · 적·유닛 스폰·이동 · 승패 조건 · 라운드/웨이브 유무 | 맵별 |
| 유닛/엔티티 | 종류 + 3D 모델 + 능력치 | 맵별 |
| **합체(합성)** | 합체 베이·합성 메커니즘 | **네모네모 전용** ✅ |
| 업그레이드 항목 | 무엇을 올릴지(공격/사거리/…)의 목록 | 맵별 ✅ |
| 자원 수급 | 수급량·배율·획득 방식 | 맵별 ✅ |
| HUD 표시 요소 | 라운드·타이머 등 개별 요소 표시 여부 | 맵별 on/off ✅ |
| 멀티 기능 | 배속 투표 · 일시정지 제한 · 플레이어 관전 · 이탈 처리 | 맵별 on/off ✅ |

## 🧩 지향 구조 (per-map config)

공통 셸이 **맵 config**를 읽어 화면을 구성한다. 예시 형태:

```js
const MAP_CONFIG = {
  nemo: {
    tabs: ['Main','Unit','Upgrade','Players'],   // 하단 탭(설정 탭은 셸이 항상 추가)
    features: { speedVote:true, pauseLimit:true, spectate:true, leave:true, rounds:true },
    hud: { round:true, timer:true, kill:true, resources:true, mapName:true },
    resources: { mineral:true, energy:true, pop:true, rate:1.0 },   // 구조 공통, 수급만 조정
    upgrades: [ /* 맵별 업그레이드 항목 목록 (골격은 공통) */ ],
    merge: true,            // 합체(네모네모 전용)
    gameplay: 'nemoModule', // 보드/적/유닛/승패 = 맵별 모듈
  },
  marine: {
    tabs: ['Main','Shop','Skills'],
    features: { speedVote:true, pauseLimit:false, spectate:true, leave:true, rounds:false },
    hud: { round:false, timer:true, kill:true, resources:true, mapName:true },
    resources: { mineral:true, energy:true, pop:false, rate:1.2 },
    upgrades: [ /* 마린 강화 항목 */ ],
    merge: false,
    gameplay: 'marineModule',
  },
  // ...
};
```

- **셸**: config의 `tabs/features/hud/resources/upgrades`를 읽어 UI를 구성.
- **게임플레이 모듈**: 보드·적·유닛·승패 등 맵 고유 로직(`gameplay`)을 맵별로 구현.
- **공통 자원/업그레이드/멀티기능**: 셸이 제공, config로 on/off·수치만 조정.

## ❓ 추후 확정 필요(미정)

- 각 신규 맵(성큰타워/마린키우기/신전뿌수기/컴퓨터가싸운다/포토겹치기)의 **게임플레이 모듈** 상세 설계.
- 맵마다 **표준 탭 세트** 권장안(완전 자유지만 자주 쓰는 조합 정의 여부).
- ‘설정·플레이어’ 탭을 셸이 항상 강제할지, 맵이 끌 수 있는지.
